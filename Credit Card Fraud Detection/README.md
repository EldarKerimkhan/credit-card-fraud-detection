# Промышленная система фрод-мониторинга кредитных карт

Решение задачи бинарной классификации для обнаружения мошеннических транзакций на экстремально несбалансированных данных (0.17% фрода). 

## 🚀 Главные достижения проекта
* **PR-AUC поднят с базовых 0.7868 до 0.8251.**
* **F1-Score увеличен в 3 раза (с 0.27 до 0.84).**
* На тестовой выборке из 85 443 транзакций модель допустила **всего 3 ложные блокировки** (`False Positive`) при успешном перехвате 75% мошенников.

## 🛠️ Инженерные решения (Production Best Practices)
1. **Временная валидация (Time-Based Split):** Данные разделены строго хронологически (70/30) без случайного перемешивания, что исключило утечку данных из "будущего" и симулирует реальный продакшн.
2. **Стоимостное обучение (Cost-Sensitive):** Вместо генерации шума через SMOTE (использовали при первом подходе и решении задачи)применены веса объектов (`sample_weight`), пропорциональные логарифму суммы (`Log_Amount`). Модель оптимизирована под спасение реальных денег, а не абстрактных строк.
3. **Временной инжиниринг:** Время переведено в циклическую шкалу (`Hour_Sin`/`Hour_Cos`), склеивая ночь. Добавлены скользящие окна по суммам и экспоненциальные средние (`EMA`) для детекции скриптовых атак.
4. **Ансамблирование:** Построен бленд моделей `CatBoost` и `LightGBM`, гиперпараметры которых автоматически подобраны байесовской оптимизацией через `Optuna`.

## 📈 Бизнес-расклад (Порог 0.91)
* Честные транзакции прошли успешно: **85 332**
* Поймано мошенников (`True Positive`): **81 из 108**
* Ошибочно заблокировано карт (`False Positive`): **3**
* Пропущено мошенников (`False Negative`): **27**
